About

Why deliberately over-engineer a personal website?

This website started as a simple portfolio but evolved into a full-scale DevOps playground where I can experiment with infrastructure patterns, automation workflows, and architectural decisions.

57 microservices · Hybrid cloud · Production-grade infrastructure
WHO I AM

I'm John Hope Dawa, a DevOps engineer who believes the best way to learn is by doing—and sometimes overdoing. This website started as a simple portfolio but evolved into a full-scale DevOps playground where I can experiment with infrastructure patterns, automation workflows, and architectural decisions I encounter in production environments.

I got into DevOps through a mix of curiosity and necessity: breaking things, fixing them, then automating the fixes. What fascinates me most is the intersection of development and operations—how infrastructure-as-code, containerization, and orchestration transform how we build and deploy software.

WHAT 'OVERENGINEERED' MEANS

The Concept

This site is deliberately over-architected. A simple static site would work fine for a portfolio, but where's the learning in that? Instead, I've built a complex, production-grade infrastructure to showcase real DevOps skills and experiment with technologies I use professionally.

The Constraints

57 Microservices: An arbitrary number that represents commitment over practicality. Currently at 2/57 (Resume API, GCP Health API), with 55 more planned.

Hybrid Cloud: Bare-metal K3s cluster (2 mini PCs + Raspberry Pi) combined with GCP Cloud Run for serverless workloads.

Infrastructure as Code: Everything defined in code—Terraform for cloud resources, Kubernetes manifests for deployments, Docker for containers.

Full Observability: Prometheus, Grafana, and monitoring built-in from day one (well, planned for day one).

HOW IT'S BUILT

Frontend: Static HTML/CSS/JavaScript served by NGINX in a Docker container, deployed to Kubernetes.

APIs: Node.js/Express microservices with MongoDB, each containerized and independently deployable.

Infrastructure: K3s cluster for on-prem workloads, GCP Cloud Run for serverless, Terraform for cloud provisioning, GitHub Actions for CI/CD.

PHILOSOPHY & PRINCIPLES
Automate Everything

If you do it twice, script it. If you script it twice, pipeline it.

Observable by Default

Metrics, logs, and traces from day one. You can't fix what you can't see.

Small, Composable Services

Each service does one thing well. Microservices taken to the extreme.

Infrastructure as Code

Version-controlled, reproducible, declarative infrastructure. No manual console clicks.

Production-Like Environments

Dev, staging, and prod should be as similar as possible. Containers make this easy.

Learn by Breaking Things

The best learning comes from failures. This is a safe space to experiment.

PROJECT TIMELINE

v0.1 — Static Site

Basic HTML/CSS portfolio. Hosted on GitHub Pages. No backend, no complexity.

v0.2 — Docker Compose

Containerized with Docker. NGINX frontend + Node.js Resume API + MongoDB. Everything running in docker-compose locally.

v0.3 — Kubernetes (K3s)

Built bare-metal K3s cluster with 3 nodes. Migrated from docker-compose to Kubernetes deployments and services.

v0.4 — Hybrid Cloud (GCP)

Added GCP Cloud Run for serverless health check API. Terraform-managed infrastructure. First hybrid cloud workload.

v0.5 — CI/CD Pipeline

GitHub Actions for automated builds. Multi-arch Docker images (amd64 + arm64). Automated pushes to Docker Hub.

v1.0 — Current (2/57 Services)

Fully functional hybrid infrastructure. Resume API and GCP Health API live. Monitoring and observability in progress.

Coming Soon

Prometheus/Grafana dashboards, Jenkins CI/CD, Ansible automation, remaining 55 microservices, full GitOps workflow.
