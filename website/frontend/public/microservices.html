The 57-Service Philosophy

Why 57 microservices? It's an arbitrary number that represents commitment over logic. This project isn't about building the minimum viable product—it's about exploring how far microservice architecture can go, learning DevOps patterns at scale, and having fun with deliberate over-engineering.

Each service is independently deployable, containerized, and has its own purpose (even if that purpose is questionable). Some are core infrastructure, others are experimental, and a few exist purely for the challenge.

SERVICE CATALOG
LIVE SERVICES (2)
Resume API
Live
Stack: Node.js + MongoDB
Endpoint: /api/resume
Repo: website/backend/resume-api/
RESTful API serving resume data from MongoDB. Automatically seeds database on startup and provides JSON-formatted resume information.
GCP Health API
Live
Stack: Node.js on GCP Cloud Run
Endpoint: /api/health
Repo: website/backend/gcp-health-api/
Serverless health check API deployed on GCP Cloud Run. Demonstrates hybrid cloud architecture with rate limiting (50 req/hr per IP). Auto-scales from 0 to 3 instances.
